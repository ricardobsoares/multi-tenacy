Visão Geral da Migração

Esta aplicação interativa resume a estratégia para transformar um sistema de inquilino único (single-tenant) em uma plataforma SaaS multi-tenant robusta e escalável, utilizando autenticação JWT segura com React e NestJS.

Autenticação Segura

Implementar o padrão Refresh Token Rotation para mitigar riscos de XSS e CSRF, garantindo sessões seguras e uma melhor experiência do usuário.

Modelo de Dados

Adotar o modelo de Banco de Dados Compartilhado com uma coluna discriminadora `tenant_id`, oferecendo o melhor equilíbrio entre custo e complexidade para a fase inicial.

Arquitetura Consciente

Tornar cada camada da aplicação, do front-end ao back-end, "consciente do tenant", assegurando o isolamento de dados em todas as operações.

O Dilema Multi-Tenant

A escolha do modelo de dados é a decisão arquitetônica mais impactante. A estratégia correta equilibra isolamento, custo e complexidade.

Comparativo dos Modelos de Banco de Dados

DB por Tenant

Isolamento Máximo

  • ✔ Segurança e performance isoladas.
  • ✔ Facilita conformidade (GDPR).
  • ✘ Custo e complexidade operacional altos.

Schema por Tenant

Equilíbrio Lógico

  • ✔ Bom isolamento lógico.
  • ✔ Custo moderado.
  • ✘ Migrações ainda complexas.
Recomendado

DB Compartilhado

Eficiência Máxima

  • ✔ Menor custo e complexidade inicial.
  • ✔ Ideal para validar o modelo de negócio.
  • ✘ Risco de vazamento de dados se mal implementado.
